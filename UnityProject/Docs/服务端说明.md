# GameNetty 服务端说明

> 本文说明当前仓库中 GameNetty 服务端目录作用、配置与协议位置、构建启动方式，以及 Unity 客户端如何联调。

---

## 一、目录结构

服务端相关目录位于仓库根目录：

```text
TEngine-main/
├── Server/                 # .NET 服务端工程
├── Config/                 # 服务端配置、协议源文件、运行时 bytes
├── Share/                  # 编译期 Analyzer / SourceGenerator / 工具
└── Bin/                    # dotnet build 输出目录，运行服务端时从这里启动
```

### Server/

```text
Server/
├── App/                    # 程序入口，Program.Main()
├── Core/                   # ET 核心运行时、Fiber、网络、日志、配置加载等
├── Loader/                 # 启动初始化、命令行解析、加载 Hotfix.dll
├── Model/                  # 配置类、消息类、组件定义、数据模型
├── Hotfix/                 # 业务逻辑：Router / Realm / Gate / Map / Watcher 等
├── ThirdParty/             # 第三方依赖封装
├── Server.sln              # 服务端解决方案
├── Start Server(Single Process).bat
└── Start Watcher.bat
```

关键入口：

```text
Server/App/Program.cs
Server/Loader/Init.cs
Server/Loader/CodeLoader.cs
```

启动链路：

```text
Program.Main()
  -> Entry.Init()
  -> Init.Start()
  -> 解析命令行 Options
  -> 初始化 Logger / TimeInfo / FiberManager
  -> CodeLoader 加载 Hotfix.dll
  -> Model.Entry.Start()
  -> 根据 AppType / Process / StartConfig 创建服务 Fiber
```

### Config/

`Config/` 是服务端运行配置和协议源数据目录。

```text
Config/
├── Excel/
│   ├── GameConfig/         # 游戏配置 Excel
│   └── StartConfig/        # 启动配置 Excel
├── Generate/               # Luban 导出的 .bytes，服务端运行时读取这里
├── Proto/                  # 网络协议 .proto
└── NLog/                   # NLog 日志配置
```

服务端运行时读取路径在：

```text
Server/Loader/ConfigLoaderInvoker.cs
```

读取规则：

```text
../Config/Generate/{ConfigName}.bytes
../Config/Generate/{StartConfig}/{ConfigName}.bytes
```

因为服务端从 `Bin/` 目录运行，所以 `Config/` 必须放在仓库根目录：

```text
TEngine-main/Config
TEngine-main/Bin
```

### Share/

`Share/` 是编译期共享工具目录。

```text
Share/
├── Analyzer/               # Roslyn Analyzer
├── Share.SourceGenerator/  # Source Generator，生成 ET System / 序列化注册等代码
└── Tool/                   # Proto2CS 等工具
```

`Server/Model.csproj`、`Server/Hotfix.csproj` 会引用这里：

```xml
<ProjectReference Include="..\..\Share\Analyzer\Share.Analyzer.csproj" OutputItemType="Analyzer" ReferenceOutputAssembly="false" />
<ProjectReference Include="..\..\Share\Share.SourceGenerator\Share.SourceGenerator.csproj" OutputItemType="Analyzer" ReferenceOutputAssembly="false" />
```

没有 `Share/` 时，服务端大概率无法完整编译。

## 二、启动配置在哪里改

启动配置源表：

```text
Config/Excel/StartConfig/
```

常用文件：

```text
StartMachineConfig.xlsx     # 机器/IP 配置
StartProcessConfig.xlsx     # 进程配置
StartSceneConfig.xlsx       # 每个进程包含哪些 Scene/Fiber
StartZoneConfig.xlsx        # 区服配置
```

当前已有配置组：

```text
Localhost
Release
Benchmark
RouterTest
```

运行脚本默认使用：

```text
StartConfig/Localhost
```

对应运行时 bytes：

```text
Config/Generate/StartConfig/Localhost/
```

服务端命令行参数：

```text
--AppType=Server|Watcher|GameTool|...
--Process=1
--StartConfig=StartConfig/Localhost
--Develop=0
--LogLevel=0
--Console=1
```

定义位置：

```text
Server/Core/Core/World/Module/Options/Options.cs
```

## 三、协议在哪里改

协议源文件：

```text
Config/Proto/
├── ClientMessage_C_1000.proto
├── OuterMessage_C_10001.proto
└── InnerMessage_S_20001.proto
```

含义：

| 文件 | 用途 |
|------|------|
| `ClientMessage_C_1000.proto` | 客户端本地/NetClient 内部消息 |
| `OuterMessage_C_10001.proto` | 客户端与外网服务通信协议 |
| `InnerMessage_S_20001.proto` | 服务端内部进程通信协议 |

协议生成工具：

```text
Share/Tool/Proto2CS/Proto2CS.cs
```

默认生成路径：

```text
客户端：../Unity/Assets/GameScripts/GameProto/Generate/Message/
服务端：../Server/Model/Generate/Message/
```

当前项目 Unity 路径是：

```text
UnityProject/Assets/GameScripts/HotFix/GameProto/Generate/Message/
```

所以后续如果重新生成协议，需要注意生成路径是否适配当前仓库结构；否则需要手动指定 `clientPath` 或同步调整工具默认路径。

当前客户端协议目录：

```text
UnityProject/Assets/GameScripts/HotFix/GameProto/Generate/Message/
```

当前服务端协议目录：

```text
Server/Model/Generate/Message/
```

## 四、构建服务端

需要安装 .NET 8 SDK/Runtime。

检查：

```powershell
dotnet --list-sdks
dotnet --list-runtimes
```

构建：

```powershell
cd E:\aaa\unity\TEngine-main
dotnet build .\Server\Server.sln
```

输出目录：

```text
E:\aaa\unity\TEngine-main\Bin
```

其中关键文件：

```text
Bin/App.exe 或 Bin/App.dll
Bin/Hotfix.dll
Bin/Hotfix.pdb
```

`Loader/CodeLoader.cs` 会从运行目录读取：

```text
./Hotfix.dll
./Hotfix.pdb
```

所以服务端必须从 `Bin/` 目录启动。

## 五、启动方式

### 单进程启动

脚本：

```text
Server/Start Server(Single Process).bat
```

等价命令：

```powershell
cd E:\aaa\unity\TEngine-main\Bin
dotnet App.dll --Process=1 --StartConfig=StartConfig/Localhost --Console=1
```

或如果存在 `App.exe`：

```powershell
App.exe --Process=1 --StartConfig=StartConfig/Localhost --Console=1
```

适合本机简单验证。

### Watcher 启动

脚本：

```text
Server/Start Watcher.bat
```

等价命令：

```powershell
cd E:\aaa\unity\TEngine-main\Bin
dotnet App.dll --AppType=Watcher --StartConfig=StartConfig/Localhost --Console=1
```

`Watcher` 是守护进程，会读取 `StartProcessConfig`，自动拉起本机配置的多个 Server 进程。

简单测试优先使用单进程脚本；需要模拟多进程部署时再用 Watcher。

## 六、客户端联调

客户端网络开关在：

```text
UnityProject/Assets/GameScripts/HotFix/GameLogic/GameApp.cs
```

```csharp
private static bool _enableNetwork = true;
```

本地只调 UI 时设为 `false`；联调服务端时设为 `true`。

当前客户端登录验证：

```csharp
await GameModule.Network.InitializeAsync();
long playerId = await ET.LoginHelper.Login(GameModule.Network.Root, "test", "123456");
Log.Info("GameNetty 登录成功，PlayerId={0}", playerId);
```

客户端默认路由地址在：

```text
UnityProject/Assets/GameScripts/HotFix/GameLogic/NetClient/Router/RouterAddressComponentSystem.cs
UnityProject/Assets/GameScripts/HotFix/GameLogic/NetClient/Main2NetClient_LoginHandler.cs
```

默认常量来自 GameNetty：

```csharp
ET.ConstValue.RouterHttpHost = "127.0.0.1";
ET.ConstValue.RouterHttpPort = 30300;
```

服务端和客户端需要使用一致的协议与端口配置。

## 七、常见问题

### 缺少 .NET 8

报错：

```text
You must install or update .NET to run this application.
Framework: Microsoft.NETCore.App, version 8.0.0
```

安装 .NET 8 SDK/Runtime 后重试。

### 找不到 Hotfix.dll

确认是否已经构建：

```powershell
dotnet build .\Server\Server.sln
```

并且从 `Bin/` 启动。

### 找不到配置 bytes

确认存在：

```text
Config/Generate/StartConfig/Localhost/
Config/Generate/*.bytes
```

并且服务端从 `Bin/` 运行。

### 客户端连接失败

检查：

- 服务端是否已启动
- `StartConfig/Localhost` 端口是否与客户端常量一致
- 防火墙是否拦截
- 客户端 `_enableNetwork` 是否为 `true`
- 客户端和服务端协议是否同步

### 修改协议后客户端/服务端不一致

修改 `Config/Proto/*.proto` 后，需要重新生成：

```text
UnityProject/Assets/GameScripts/HotFix/GameProto/Generate/Message/
Server/Model/Generate/Message/
```

并重新构建服务端、热更 DLL。
