# 3-8. 网络模块 - NetworkModule

## 🌐 模块概述

网络模块是 TEngine 框架提供的**通用 HTTP 通信模块**，基于 `UnityWebRequest` + `UniTask` 实现，适用于 REST API 调用、JSON 数据交互、文件下载等常见客户端网络场景。

### ✨ 核心优势

1. **异步优先**
   - 全部 API 返回 `UniTask`，不阻塞主线程
   - 支持 `CancellationToken` 取消单个请求

2. **统一响应结构**
   - `NetworkResponse` / `NetworkResponse<T>` 封装成功/失败/超时/取消状态
   - 内置 JSON 序列化/反序列化（基于 `Utility.Json`）

3. **开箱即用**
   - 支持 `BaseUrl` 相对路径拼接
   - 支持 `AuthToken` 自动附加 Bearer 鉴权头
   - 支持全局默认请求头

4. **模块体系集成**
   - 通过 `GameModule.Network` 访问
   - 遵循 TEngine 模块命名约定，首次访问自动创建

> **说明**：本模块面向 HTTP/REST 场景。若需要 WebSocket、Protobuf、长连接等实时通信，建议结合 [Fantasy](https://github.com/Quan666/Fantasy)、[GameNetty](https://github.com/ALBERT729/GameNetty) 等方案自行扩展，或在 `NetworkModule` 基础上封装业务层协议。

## ⚡ 快速开始

### 1. 配置基础地址

建议在 `GameApp.StartGameLogic()` 或登录流程初始化时配置：

```csharp
GameModule.Network.BaseUrl = "http://127.0.0.1:8080";
GameModule.Network.DefaultTimeout = 15f;
```

### 2. 发起 GET 请求

```csharp
NetworkResponse response = await GameModule.Network.GetAsync("/api/version");

if (response.IsSuccess)
{
    Log.Info("版本信息: {0}", response.Text);
}
else
{
    Log.Error("请求失败: {0}", response.Error);
}
```

### 3. 发起 JSON POST 请求

```csharp
[Serializable]
public class LoginReq
{
    public string Account;
    public string Password;
}

[Serializable]
public class LoginResult
{
    public int Code;
    public string Token;
    public string Message;
}

var req = new LoginReq { Account = "test", Password = "123456" };
NetworkResponse<LoginResult> response = await GameModule.Network
    .PostJsonAsync<LoginReq, LoginResult>("/api/login", req);

if (response.IsSuccess)
{
    GameModule.Network.AuthToken = response.Result.Token;
    Log.Info("登录成功");
}
```

## 📖 核心 API

### 访问入口

```csharp
GameModule.Network   // INetworkModule
```

### 配置属性

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `BaseUrl` | `string` | `""` | 服务器基础地址，如 `http://127.0.0.1:8080` |
| `DefaultTimeout` | `float` | `10f` | 默认超时时间（秒） |
| `AuthToken` | `string` | `""` | 鉴权 Token，设置后自动附加 `Authorization: Bearer {token}` |

### 请求头管理

```csharp
GameModule.Network.SetDefaultHeader("X-Platform", "Android");
GameModule.Network.SetDefaultHeader("X-Version", "1.0.0");
GameModule.Network.RemoveDefaultHeader("X-Platform");
GameModule.Network.ClearDefaultHeaders();
```

### HTTP 方法

| 方法 | 说明 |
|------|------|
| `GetAsync(url, options?, ct?)` | GET 请求 |
| `PostAsync(url, body, options?, ct?)` | POST 原始文本 |
| `PostFormAsync(url, formFields, options?, ct?)` | POST 表单 |
| `PutAsync(url, body, options?, ct?)` | PUT 请求 |
| `DeleteAsync(url, options?, ct?)` | DELETE 请求 |
| `GetJsonAsync<T>(url, options?, ct?)` | GET + JSON 反序列化 |
| `PostJsonAsync<TRequest>(url, data, options?, ct?)` | POST JSON 请求体 |
| `PostJsonAsync<TRequest, TResponse>(url, data, options?, ct?)` | POST JSON 请求 + 反序列化响应 |
| `DownloadBytesAsync(url, options?, ct?)` | 下载二进制数据 |
| `DownloadFileAsync(url, savePath, options?, ct?)` | 下载文件到本地 |
| `CancelAllRequests()` | 取消所有进行中的请求 |

## 📦 NetworkResponse 响应结构

### NetworkResponse

| 字段 | 类型 | 说明 |
|------|------|------|
| `IsSuccess` | `bool` | 请求是否成功 |
| `StatusCode` | `long` | HTTP 状态码 |
| `Text` | `string` | 响应文本 |
| `Data` | `byte[]` | 响应二进制数据 |
| `Error` | `string` | 错误信息 |
| `IsCanceled` | `bool` | 是否被取消 |
| `IsTimeout` | `bool` | 是否超时 |
| `Url` | `string` | 实际请求 URL |

### NetworkResponse\<T\>

继承 `NetworkResponse`，额外包含：

| 字段 | 类型 | 说明 |
|------|------|------|
| `Result` | `T` | JSON 反序列化结果 |

## 🔧 NetworkRequestOptions 请求参数

单次请求可通过 `NetworkRequestOptions` 覆盖默认行为：

```csharp
var options = new NetworkRequestOptions
{
    Timeout = 30f,
    ContentType = "application/json; charset=utf-8",
    Headers = new Dictionary<string, string>
    {
        { "X-Request-Id", Guid.NewGuid().ToString() }
    }
};

await GameModule.Network.GetAsync("/api/data", options);
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `Timeout` | `float?` | 单次请求超时（秒），为空时使用 `DefaultTimeout` |
| `Headers` | `Dictionary<string, string>` | 额外请求头（与默认头合并，同名以本次为准） |
| `ContentType` | `string` | POST/PUT 请求 Content-Type |
| `SkipDefaultContentType` | `bool` | 是否跳过自动设置 Content-Type |

## 💡 使用示例

### URL 拼接规则

```csharp
GameModule.Network.BaseUrl = "http://127.0.0.1:8080";

// 相对路径 → http://127.0.0.1:8080/api/user
await GameModule.Network.GetAsync("/api/user");

// 相对路径（无斜杠）→ http://127.0.0.1:8080/api/user
await GameModule.Network.GetAsync("api/user");

// 绝对路径 → 直接使用，不拼接 BaseUrl
await GameModule.Network.GetAsync("https://cdn.example.com/config.json");
```

### 表单 POST

```csharp
var form = new Dictionary<string, string>
{
    { "username", "test" },
    { "password", "123456" }
};

NetworkResponse response = await GameModule.Network.PostFormAsync("/api/login", form);
```

### 文件下载

```csharp
string savePath = System.IO.Path.Combine(Application.persistentDataPath, "patch.zip");
NetworkResponse response = await GameModule.Network.DownloadFileAsync(
    "/api/download/patch",
    savePath);

if (response.IsSuccess)
{
    Log.Info("下载完成: {0}", savePath);
}
```

### 请求取消

```csharp
using var cts = new CancellationTokenSource();

// 5 秒后自动取消
cts.CancelAfter(TimeSpan.FromSeconds(5));

NetworkResponse response = await GameModule.Network.GetAsync("/api/slow", cancellationToken: cts.Token);

if (response.IsCanceled)
{
    Log.Warning("请求已取消");
}
```

### UI 中发起网络请求

```csharp
private async void OnClickLoginBtn()
{
    m_btnLogin.interactable = false;

    try
    {
        var req = new LoginReq
        {
            Account = m_inputAccount.text,
            Password = m_inputPassword.text
        };

        var response = await GameModule.Network
            .PostJsonAsync<LoginReq, LoginResult>("/api/login", req);

        if (response.IsSuccess)
        {
            GameModule.Network.AuthToken = response.Result.Token;
            // 进入游戏...
        }
        else
        {
            Log.Error("登录失败: {0}", response.Error);
        }
    }
    finally
    {
        m_btnLogin.interactable = true;
    }
}
```

### 封装业务网络层（推荐）

建议在热更层 `GameLogic` 中封装业务 API，避免 UI 直接拼接 URL：

```csharp
public static class LoginService
{
    public static async UniTask<NetworkResponse<LoginResult>> LoginAsync(string account, string password)
    {
        return await GameModule.Network.PostJsonAsync<LoginReq, LoginResult>(
            "/api/login",
            new LoginReq { Account = account, Password = password });
    }
}
```

## ⚠️ 注意事项

### 1. JSON 序列化限制

默认使用 `Utility.Json`（底层为 `JsonUtility`），要求：

- 请求/响应类型标记 `[Serializable]`
- 仅支持字段（field），不支持属性（property）
- 不支持 Dictionary、多态等复杂结构

如需 Newtonsoft 等高级序列化，可替换 `RootModule` 中的 `jsonHelperTypeName`。

### 2. 与 Utility.Http 的关系

框架中还存在 `Utility.Http` 工具类（含部分 Coroutine 实现）。**新业务请优先使用 `GameModule.Network`**，具备统一响应结构、鉴权、超时与取消能力。

### 3. 生命周期

- 模块在首次访问 `GameModule.Network` 时自动创建
- 场景切换时模块实例保持，配置（BaseUrl、Token 等）不会自动重置
- 退出游戏时调用 `GameModule.Shutdown()` 清空模块缓存引用
- UI 关闭时如有进行中的请求，建议通过 `CancellationToken` 取消，或在适当时机调用 `CancelAllRequests()`

### 4. 热更边界

- `INetworkModule`、`NetworkModule` 位于 `TEngine.Runtime`（AOT 程序集），热更代码通过 `GameModule.Network` 调用
- 业务协议结构体/DTO 定义在 `GameProto` 或 `GameLogic` 热更程序集中

### 5. 平台差异

- 编辑器 / 移动端 / WebGL 均使用 `UnityWebRequest`
- WebGL 需注意跨域（CORS）与服务端响应头配置
- HTTPS 证书问题需在服务端或平台层处理

## ❌ 常见错误

| 错误写法 | 正确写法 | 原因 |
|---------|---------|------|
| `ModuleSystem.GetModule<INetworkModule>()` | `GameModule.Network` | 应通过 GameModule 统一访问 |
| 相对路径但未设置 `BaseUrl` | 先设置 `BaseUrl` 或使用完整 URL | 相对路径依赖 BaseUrl 拼接 |
| JSON 类使用 property | 使用 public field + `[Serializable]` | JsonUtility 不支持 property |
| UI 销毁后仍等待网络回调 | 传入 `CancellationToken`，在 `OnDestroy` 取消 | 避免闭包引用已销毁 UI |
| 每次请求手动拼 Authorization | 设置 `GameModule.Network.AuthToken` | 模块自动附加 Bearer 头 |
| 使用 `Utility.Http.Get`（Coroutine） | `await GameModule.Network.GetAsync` | 新代码应使用 UniTask 异步 API |
