# 3-8. 网络模块 - GameNetty

## 模块概述

网络模块已接入 [GameNetty](https://github.com/Alex-Rachel/GameNetty/tree/main)，替换原先基于 `UnityWebRequest` 的 HTTP 工具型实现。GameNetty 是从 ET 客户端/服务端体系拆分出的前后端分离网络方案，客户端侧包含 ET Core、ET Network、MemoryPack 协议序列化、KCP/UDP 与 WebSocket 传输。

当前项目保留 TEngine 的统一访问入口：

```csharp
GameModule.Network   // INetworkModule
```

`NetworkModule` 负责初始化 GameNetty 运行时、驱动 `TimeInfo` / `FiberManager`、暴露当前 `Root` 和 `Session`，业务登录与协议调用放在热更层 `GameLogic/NetClient`。

```text
    └── GameLogic/NetClient/                 # GameNetty 客户端登录、路由、心跳、发送器

```

## 启动流程

`GameApp.StartGameLogic()` 会先初始化网络模块：

```csharp
private static async void StartGameLogic()
{
    await GameModule.Network.InitializeAsync();

    GameModule.UI.ShowUIAsync<LoginUI>();
}
```

初始化时会完成：

1. 注册 ET Logger / TimeInfo / FiberManager。
2. 扫描 `TEngine.Runtime`、`ET.Core`、`ET.Network`、`GameLogic`、`GameProto` 中的 ET 特性类型。
3. 调用 `ET.Entry.Start()` 创建主 Fiber。
4. 在 `AppStartInitFinish` 事件中保存 GameNetty `Root`。
5. 由 TEngine `IUpdateModule` 每帧驱动 ET `Update/LateUpdate`。

## 核心 API

### 初始化

```csharp
await GameModule.Network.InitializeAsync();
```

通常由 `GameApp` 统一调用，业务代码无需重复初始化。

### 获取 Root / Session

```csharp
ET.Scene root = GameModule.Network.Root;
ET.Session session = GameModule.Network.Session;
```

`Root` 初始化后可用于 GameNetty 组件操作；`Session` 需要登录成功后才存在。

### 登录

GameNetty 示例登录逻辑位于热更层：

```csharp
await ET.LoginHelper.Login(GameModule.Network.Root, account, password);
```

默认路由配置在：

```csharp
ET.ConstValue.RouterHttpHost = "127.0.0.1";
ET.ConstValue.RouterHttpPort = 30300;
```

登录流程会通过 Router 获取 Realm/Gate 地址，创建 Gate Session，并写入 `SessionComponent`。

### 发送消息

```csharp
C2G_Ping ping = C2G_Ping.Create();
GameModule.Network.Send(ping);
```

### 请求响应

```csharp
C2G_Ping request = C2G_Ping.Create();
G2C_Ping response = await GameModule.Network.CallAsync<G2C_Ping>(request);

if (response.Error == 0)
{
    Log.Info("Ping 成功");
}
```

### 断开连接

```csharp
GameModule.Network.Disconnect();
```

`GameApp.Release()` 中会调用 `Disconnect()`，模块 `Shutdown()` 时会释放 ET `World`。

## 协议与业务层

当前导入的是 GameNetty 示例协议：

```text
GameProto/Generate/Message/
├── ClientMessage_C_1000.cs
└── OuterMessage_C_10001.cs
```

实际项目接入服务端时，应根据服务端协议重新生成 `GameProto/Generate/Message`，并保持前后端 opcode、MemoryPack 字段顺序一致。

业务请求建议封装在热更层，例如：

```csharp
public static class NetService
{
    public static async UniTask<G2C_Ping> PingAsync()
    {
        C2G_Ping request = C2G_Ping.Create();
        return await GameModule.Network.CallAsync<G2C_Ping>(request);
    }
}
```

## 平台说明

- 非 WebGL 默认使用 UDP/KCP。
- WebGL 使用 `UnityWebSocket`，GameNetty 代码中通过 `#if UNITY_WEBGL` 自动切换。
- 服务器路由地址由 `ConstValue.RouterHttpHost` / `RouterHttpPort` 控制。

---

## 注意事项

1. `GameModule.Network.InitializeAsync()` 必须在调用 `LoginHelper.Login`、`Send`、`CallAsync` 前完成。
2. `GameProto` 消息依赖 MemoryPack，Unity 首次打开项目后需要解析 `Packages/manifest.json`。
3. `TEngine.Runtime` 只负责网络运行时门面，不强引用热更协议登录实现，避免 AOT 层反向依赖 `GameLogic/GameProto`。
4. 如果替换协议文件，请同步重新生成 HybridCLR 热更 DLL。
5. 原 `NetworkResponse` / `NetworkRequestOptions` HTTP API 已移除，`INetworkModule` 主 API 改为 GameNetty 初始化、Session 发送与请求。

## 常见错误

| 错误 | 原因 | 修复 |
|------|------|------|
| `GameNetty network module is not initialized.` | 未初始化网络模块 | 先 `await GameModule.Network.InitializeAsync()` |
| `GameNetty session is invalid. Please login first.` | 未登录或 Session 已断开 | 先调用 `ET.LoginHelper.Login(...)` |
| 找不到 `MemoryPack` | 包依赖未解析 | 让 Unity 重新解析 `Packages/manifest.json` |
| 协议反序列化失败 | 前后端消息结构/opcode 不一致 | 重新生成 `GameProto/Generate/Message` |
| WebGL 无法连接 | WebGL 不支持 UDP/KCP | 使用 WebSocket 服务端入口 |
